// Implementazione Thread-Safe
public void run() {
  while(true) {
    pensare(); 

    // Scelta dell'ordine delle bacchette
    // per evitare l'attesa circolare (Deadlock)
    int primo = id;
    int secondo = (id + 1) % 5;

    // Se è l'ultimo filosofo, inverte l'ordine
    if (id == 4) {
        primo = (id + 1) % 5;
        secondo = id;
    }

    synchronized(forks[primo]) {
      synchronized(forks[secondo]) {
        mangiare(); // Sezione Critica
      }
    }
  }
}

Gestione del Conflitto

La simulazione utilizza una gerarchia delle risorse per prevenire il deadlock.

  • ✔ Mutua Esclusione: Solo un filosofo può tenere una bacchetta.
  • ✔ No Deadlock: L'ultimo filosofo (ID 4) rompe la simmetria prendendo le bacchette in ordine inverso.
